# PulsePeak — Product Craftsmanship Directive

**Status: GOVERNING (owner, 2026-07-10). Supersedes and absorbs the Cinematic
Experience Directive. This is the quality bar for every screen.**

The owner is not building software. The owner is building a world-class premium
consumer product. When PulsePeak opens the reaction must be "wow" / "I want to
use this" / "this feels expensive" — never "nice dashboard."

## The mental-model change (binding on all future work)
STOP thinking page → section → card → button (UI engineering; enterprise).
START thinking like an industrial designer / creative director / film director.
The job is not arranging interface components. The job is **creating desire**.

## Composition law — every screen must tell a story
- ONE clear purpose · ONE emotional direction · ONE hero · ONE supporting
  story · ONE obvious action.
- Everything else quietly supports. Nothing competes. Nothing exists just to
  fill space.
- The application must never feel ASSEMBLED (cards, cards, cards — rectangles
  of information). Ask "what experience should the user have here?", never
  "where should I place another card?"

## THE LAYOUT IS SACRED (new, hard rule)
The layout IS the brand: composition, spacing, hierarchy, interaction model.
**Themes must NOT redesign or re-architect the app.** Themes change ONLY:
materials, lighting, colour, surface treatment, atmosphere. Same luxury hotel
at sunrise vs sunset — never a different building.

## Pillars (unchanged from the Cinematic directive, now with this bar)
- **Photography IS the product** — large, cinematic, one premium commercial
  production; images are part of the experience, not content in rectangles.
- **Anatomy teaches** — premium interactive body models; muscles illuminate;
  never clip-art or functional diagrams.
- **Motion** — nothing appears; everything emerges. Cards breathe, photography
  subtly moves, progress fills, buttons feel physical. Never flashy. "This
  feels expensive."
- **Surfaces** — material, depth, light, texture, atmosphere. Never CSS boxes,
  never database records.
- **Moments** — Today opening should make them smile; finishing a workout,
  hitting a PR, opening Progress should each have a crafted, memorable moment.

## The standard (screen-complete test)
Would Apple ship this? Would Nike publish it? Would WHOOP charge for it?
Would Arc'teryx billboard it? **Would someone screenshot it to show a friend
because it looks beautiful?** If not — keep refining.

## Still binding underneath
Honesty (no fake data/media; entitlement and renewal terms stay visible),
accessibility (reduced-motion, AA both themes), performance (GPU-friendly
motion, first-load budget).
